# 4. Нефункциональные требования

Здесь описано, как система должна работать с технической точки зрения

## Производительность
- Система должна принимать события без блокировки
  основного потока обработки запросов
- Время ответа API не должно зависеть от времени
  выполнения фоновых задач

## Надёжность и доставка сообщений
- Система должна обеспечивать доставку событий
  как минимум один раз
- Потеря событий при сбоях компонентов недопустима

## Обработка сбоев
- Повторная обработка одного и того же события
  не должна приводить к некорректному состоянию данных
- Система должна корректно завершать обработку сообщений
  при остановке сервисов

## Наблюдаемость
- Система должна обеспечивать логирование ключевых событий обработки
- Должны быть доступны метрики обработки сообщений

## Безопасность
- Доступ к внутренним API системы должен быть ограничен
- Данные пользователей должны обрабатываться в соответствии
  с базовыми требованиями безопасности

## Масштабируемость
- Система должна поддерживать горизонтальное масштабирование
- Обработка событий должна распределяться между экземплярами сервисов
