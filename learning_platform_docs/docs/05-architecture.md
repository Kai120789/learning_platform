# 5. Архитектура приложения

Здесь описывается, какие сервисы будут реализованы в рамках системы и их кратко описанный функционал

## Сервис авторизации

Отвечает за авторизацию пользователя, работу с JWT токенами и сессиями, для сохранения пользователя при регистрации отправляет запрос на сервис пользователей

## Сервис уведомлений

Создает уведомления о событии, отправляет запросы на сервис бота для отправки уведомлений

## Сервис уроков

Создание и редактирование тестов, уроков, домашних заданий. Также сервис отвечает за загрузку вебинаров и других медиа, которые получает в S3

## Сервис пользователей

Отвечает за работу с пользователями и их группами. Создание пользователей, редактирование данных, их настроек, а также работу с группами (создание группы, добавление пользователя)

## Сервис расписаний

Сервис работает с расписаниями, а именно отлавливает событие и асинхронно обновляет расписания для всех пользователей, что важно для согласованности данных

## Сервис интерактивной доски

Сервис отвечает за совместное использование интерактивной доски в реальном времени с использованием веб-сокетов

## Телеграм бот

Отправляет уведомления в группы, куда будет добавлен бот, либо лично пользователю

## Сервис моделей (библиотека)

Сервис для работы с сущностями, описанными в бд, а также с миграциями

## API Gateway

Единая точка входа, проксирует запросы на разные бэкенд сервисы, отвечает за проверку авторизации

## Postgres

Хранение структурированных данных данных

## Mongo

Хранение данных без четкой структуры (тестов, дз, уроков)

## Kafka

Брокер сообщений для обработки событий

## Redis

Хранение токенов

# S3 (Minio)

Хранение медиа контента (вебинары, видео, фото)
