# 5. Архитектура приложения

Здесь описывается, какие сервисы будут реализованы в рамках системы и их кратко описанный функционал

## Сервис авторизации

Отвечает за авторизацию пользователя, работу с JWT токенами и сессиями, для сохранения пользователя при регистрации отправляет запрос на сервис пользователей

## Сервис уведомлений

Создает уведомления о событии, отправляет запросы на сервис бота (через брокер) для отправки уведомлений

## Сервис уроков

Создание и редактирование онлайн уроков/вебинаров, привязка ссылки на звонок, интерактивной доски

## Сервис пользователей

Отвечает за работу с пользователями. Создание пользователей, редактирование данных, их настроек

## Сервис групп

Отвечает за работу с группами (создание, редактирование, удаление), а также привязку пользователей к группам

## Сервис расписаний

Сервис отвечает за управление слотами, рабочими графиками преподавателей, расписанием групп и пользователей, а также проверку конфликтов при планировании занятий

## Сервис учебных программ/курсов

Сервис структурирует блоки с теорией и практическими заданиями и собирает из них полноценные курсы, доступные либо конкретному пользователю, либо группе (всем пользователям группы)

## Сервис практических заданий

Сервис позволяет создавать и редактировать шаблоны с практическими заданиями (для переиспользования), а также на их основании назначать их пользователю или группе, выбирая время с которого оно будет доступно и дедлайн.
Помимо этого практического задания это по сути всегда тест, который в асинхронно или синхронно проверяется после отправки ответов на задание

## Сервис преподавателей

Сервис отвечает за хранение информации о преподавателе (пользователь с ролью tutor), а также поиск учениками преподавателей по их требованиям.
Помимо этого сервис также предоставляет возможность создавать отзывы на преподавателей с оценкой и получать информацию о их форматах работы с ценами

## Сервис предметов

Сервис по сути отвечает только за работу с предметами, а именно их хранение и получение другими сервисами

## Сервис интерактивной доски

Сервис отвечает за совместное использование интерактивной доски в реальном времени с использованием веб-сокетов

## Сервис медиа

Отдельный сервис, отвечающий за работу с с3

## Телеграм бот

Отправляет уведомления в группы, куда будет добавлен бот, либо лично пользователю

## API Gateway

Единая точка входа, проксирует запросы на разные бэкенд сервисы, отвечает за проверку авторизации

---

## Postgres

Хранение структурированных данных (под каждый сервис пока планируется своя бд в одном постгресе)

## Mongo

Хранение данных без четкой структуры (контент интерактивной доски, скриншоты состояний)

## Kafka

Брокер сообщений для обработки событий

## Redis

Хранение токенов

## S3 (Minio)

Хранение медиа контента (вебинары, видео, фото)
